Arena++ System Overview

A premium, full-stack C++ competitive programming platform with real-time judging and a high-performance developer experience.

Introduction

Arena++ is a specialized online judge designed for high-performance C++ challenges. It provides a Raycast-inspired, distraction-free environment for developers to master algorithms and data structures.

Fast Execution

Native G++ 17 compilation on a high-concurrency Node.js backend.

Real-time Feedback

Instant verdicts with pass/fail counts for hidden test cases.

Premium UI

Minimalist, accessible design system focused on focus and clarity.

Tech Stack

  • Frontend: Vanilla JS, CSS3 (Modular System), HTML5
  • Backend: Node.js, Express.js
  • Compiler: G++ (MinGW/GCC) supporting C++17
  • Storage: High-performance local JSON-based persistence

System Architecture

/root
├── server.js (Entry Point)
├── controllers/ (Logic layer for submissions & problems)
├── routes/ (API Endpoint definitions)
├── utils/ (Judgement engine: compile, execute, compare)
├── data/ (JSON storage for problems and testcases)
└── public/ (Static frontend assets)
    ├── css/ (Base, Layout, Components, Animations)
    ├── js/ (Shared UI components)
    └── *.html (Core views)

API Documentation

Problems

GET /api/problems - Returns an array of all available challenges.

GET /api/problems/:id - Retrieves detailed metadata for a specific problem.

Submissions

POST /api/submit - Submit code for evaluation.

// Request Body
{
  "code": "string",
  "problemId": "string"
}

// Response
{
  "verdict": "Accepted",
  "passed": 12,
  "total": 12,
  "error": null
}

Judging Flow

  1. Submission: The user sends C++ code via the POST endpoint.
  2. Compilation: Backend creates a temporary .cpp file and runs g++ to generate an executable.
  3. Execution: The executable runs against stored hidden test cases within strict time/memory constraints.
  4. Comparison: Standard output is captured and compared with expected output (ignoring trailing whitespace/newlines).
  5. Verdict: Results (AC, WA, TLE, CE) are returned to the frontend.

Local Setup

Ensure you have node and g++ installed and available in your system's PATH.

# 1. Clone the repository
git clone https://github.com/OmkarPujeri/cpp-judge.git

# 2. Install dependencies
npm install

# 3. Start the server
node server.js
